# runbooks

Esta pasta guarda procedimentos de investigacao.

## Diferenca entre knowledge, runbook e playbook

- `knowledge/`: conhecimento de referencia e contexto
- `runbooks/`: como investigar um problema
- `playbooks/`: como agir ou responder de forma padronizada

## O que faz sentido guardar aqui

- checklist de troubleshooting
- comandos de validacao
- sinais que confirmam ou descartam hipoteses
- ordem recomendada de investigacao

## Regra pratica

Se a pergunta for "como eu investigo isso?", o conteudo provavelmente pertence a um runbook.
