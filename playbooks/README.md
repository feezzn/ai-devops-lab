# playbooks

Esta pasta guarda respostas padronizadas para incidentes e falhas conhecidas.

## Quando usar

Use um playbook quando a pergunta nao for mais "como investigar?", e sim:

- como conter a falha
- como restaurar o servico
- como executar rollback
- como agir de forma segura e repetivel

## Exemplos de playbooks

- rollback de deployment
- rotacao de credenciais
- reexecucao segura de pipeline
- resposta a falha de autenticacao

## Regra pratica

Se a pergunta for "como agir?", o conteudo provavelmente pertence a um playbook.
